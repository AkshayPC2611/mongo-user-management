# MongoDB User Management Console: A Modern Web-Based Solution for Database Administration

## Introduction

Managing MongoDB users and roles can be a complex task, especially when dealing with multiple databases and varying permission levels. Traditional command-line approaches, while powerful, can be time-consuming and error-prone for teams that need quick, visual access to user management functions.

Today, we're excited to introduce our **MongoDB User Management Console** - a modern, web-based GUI application that simplifies MongoDB user administration through an intuitive interface. This tool bridges the gap between MongoDB's powerful authentication system and the need for accessible, team-friendly database management.

## What is MongoDB User Management Console?

The MongoDB User Management Console is a full-stack web application built with Node.js and AngularJS that provides a comprehensive interface for managing MongoDB built-in users. It offers a secure, modern alternative to command-line user management with features designed for both database administrators and development teams.

## Installation and Setup Guide

### Prerequisites
- Node.js (v14 or higher)
- MongoDB instance (local or remote)
- npm package manager

### Step-by-Step Installation

1. **Navigate to the project directory:**
   ```bash
   cd website
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure environment variables (optional):**
   Create a `.env` file with:
   ```
   DEFAULT_ADMIN_EMAIL=admin@yourcompany.com
   DEFAULT_ADMIN_USERNAME=admin
   DEFAULT_ADMIN_PASSWORD=your_secure_password
   PORT=3000
   ```

4. **Start the application:**
   ```bash
   npm start
   ```

5. **Access the console:**
   Open your browser and navigate to `http://localhost:3000`

## User Guide

### Getting Started

**1. Authentication**
- Create an account using the signup form
- Login with your credentials to access MongoDB features
- The system includes a default admin user for initial setup

**2. MongoDB Connection**
- Navigate to the MongoDB section after authentication
- Enter your MongoDB connection string
- Supported formats:
  - Local: `mongodb://localhost:27017`
  - Authenticated: `mongodb://username:password@localhost:27017`
  - Atlas: `mongodb+srv://username:password@cluster.mongodb.net`
- Click "Connect" to establish the connection

**3. User Management Operations**

**Creating Users:**
- Click "Create User" button
- Fill in username, password, and database
- Assign roles (comma-separated): `readWrite,dbAdmin`
- Common roles include: read, readWrite, dbAdmin, userAdmin

**Managing Existing Users:**
- View all users in a paginated list
- Search users by username or roles
- Edit user passwords and role assignments
- Rename users with the rename function
- Delete users with confirmation dialog

**Advanced Role Management:**
- Access role management through "Manage Roles" button
- Create custom roles with JSON-defined privileges
- Assign built-in MongoDB roles
- Grant/revoke roles for existing users

### MongoDB Connection Examples

**Local Development:**
```
mongodb://localhost:27017
```

**Production with Authentication:**
```
mongodb://dbuser:password@production-server:27017/myapp
```

**MongoDB Atlas Cloud:**
```
mongodb+srv://username:password@cluster0.mongodb.net/myapp
```

## Common MongoDB Roles Reference

### Database-Level Roles
- **read**: Read-only access to database collections
- **readWrite**: Read and write access to database collections
- **dbAdmin**: Administrative tasks (indexing, stats, etc.)
- **dbOwner**: Full administrative access to database
- **userAdmin**: Create and modify users and roles

### Cluster-Level Roles
- **clusterAdmin**: Full cluster administrative access
- **clusterManager**: Manage and monitor cluster
- **clusterMonitor**: Read-only access to monitoring tools
- **backup**: Backup database operations
- **restore**: Restore database operations

### Any Database Roles
- **readAnyDatabase**: Read access to all databases
- **readWriteAnyDatabase**: Read/write access to all databases
- **userAdminAnyDatabase**: User management across all databases
- **dbAdminAnyDatabase**: Database administration across all databases

### Super User Role
- **root**: Unrestricted access to all operations

## Security Best Practices

### Application Security
- All passwords are hashed using bcrypt with 12 salt rounds
- Session-based authentication prevents unauthorized access
- Input validation prevents injection attacks
- Connection strings are sanitized in logs

### MongoDB Security Recommendations
- Use strong passwords for all MongoDB users
- Implement role-based access control (RBAC)
- Grant minimum necessary permissions
- Regularly audit user permissions
- Use MongoDB Atlas for enhanced cloud security

### Production Deployment
- Use HTTPS in production environments
- Configure proper firewall rules
- Set up MongoDB authentication
- Use environment variables for sensitive configuration
- Implement regular security audits

## API Reference

The application provides RESTful API endpoints for programmatic access:

### Authentication Endpoints
- `POST /api/auth/signup` - Create new account
- `POST /api/auth/login` - User login
- `POST /api/auth/logout` - User logout
- `GET /api/me` - Get current user info

### MongoDB Management Endpoints
- `POST /api/mongo/connect` - Connect to MongoDB
- `POST /api/mongo/disconnect` - Disconnect from MongoDB
- `GET /api/mongo/status` - Check connection status
- `GET /api/mongo/users` - List all users
- `POST /api/mongo/users` - Create new user
- `PUT /api/mongo/users/:username` - Update user
- `DELETE /api/mongo/users/:username` - Delete user
- `PATCH /api/mongo/users/:username/roles` - Grant roles
- `DELETE /api/mongo/users/:username/roles` - Revoke roles
- `PATCH /api/mongo/users/:username/rename` - Rename user

### Role Management Endpoints
- `GET /api/mongo/roles` - List roles in database
- `POST /api/mongo/roles` - Create custom role
- `DELETE /api/mongo/roles/:rolename` - Delete custom role

## Troubleshooting Guide

### Common Connection Issues

**Problem: "Connection refused"**
- Solution: Ensure MongoDB service is running
- Check if MongoDB is listening on the correct port
- Verify network connectivity

**Problem: "Authentication failed"**
- Solution: Verify username/password in connection string
- Ensure the connecting user has appropriate permissions
- Check if authentication is enabled on MongoDB

**Problem: "Permission denied"**
- Solution: Grant userAdmin or userAdminAnyDatabase roles
- For role management, ensure dbAdmin permissions
- Verify database-specific permissions

### Application Issues

**Problem: Cannot create users**
- Solution: Ensure connected user has userAdmin role
- Check if target database exists
- Verify role names are correct

**Problem: UI not loading**
- Solution: Check browser console for JavaScript errors
- Ensure all dependencies are installed
- Verify server is running on correct port
